    GHGA-Metadata-Schema schema¶

    The metadata schema for the German Human Genome-Phenome Archive (GHGA).

    Classes¶

    • Attribute - A key/value pair that further characterizes an entity.
    • DataUseCondition - Data Use Condition represents the use conditions associated with a Dataset. A permission field can have one or more terms that collectively defines the data use condition. The modifier determines the interpretation of the use permission(s).
    • NamedThing - A databased entity, concept or class. This is a generic class that is the root of all the other classes.
      • Agent - An agent is something that bears some form of responsibility for an activity taking place, for the existence of an entity, or for another agent's activity. Agents include a Person, Organization, or Software that performs an activity.
      • BiologicalQuality - A biological quality is a quality held by a biological entity.
        • DiseaseOrPhenotypicFeature - Disease or Phenotypic Feature that the entity is associated with. This entity is a union of Disease and Phenotypic Feature and exists to accommodate situations where Disease concepts are used interchangeably with Phenotype concepts or vice-versa.
          • Disease - A disease is a disposition to undergo pathological processes that exists in an organism because of one or more disorders in that organism.
          • PhenotypicFeature - The observable form taken by some character (or group of characters) in an individual or an organism, excluding pathology and disease. The detectable outward manifestations of a specific genotype.
      • Committee - A group of people organized for a specific purpose.
        • DataAccessCommittee - A group of members that are delegated to grant access to one or more datasets after ensuring the minimum criteria for data sharing has been met, and request for data use does not raise ethical and/or legal concerns.
      • InformationContentEntity - A generically dependent continuant that is about some thing.
        • DataAccessPolicy - A Data Access Policy specifies under which circumstances, legal or otherwise, a user can have access to one or more Datasets belonging to one or more Studies.
        • Dataset - A Dataset is a collection of Files that is prepared for distribution.
          • AggregateDataset - An Aggregate Dataset is a specialized dataset that is built by combining one or more Datasets together.
          • AnalysisDataset - An Analysis Dataset is a collection of Files generated from one or more Analysis performed on data from one or more Studies.
          • ExperimentDataset - An Experiment Dataset is a collection of Files linked to one or more Experiments from one or more Studies.
        • File
        • Protocol - A plan specification which has sufficient level of detail and quantitative information to communicate it between investigation agents, so that different investigation agents will reliably be able to independently reproduce the process.
        • Publication - The Publication entity represents a publication. While a publication can be any article that is published, the minimum expectation is that the publication has a valid DOI.
        • Technology - A Technology is an abstraction that represents the instrument used for an assay. The Technology entity captures instrument-specific attributes that are relevant for an Experiment entity. The Technology entity may be further characterized by its children where each child has fields that are relevant to that particular technology.
        • Workflow - A Workflow is an abstraction that represents the workflow used to perform an analysis. The Workflow entity captures workflow-specific attributes that are relevant for an Analysis entity. The Workflow entity may be further characterized by its children where each child has fields that are relevant to that particular workflow.
        • WorkflowStep - A Workflow Step represents each individual step performed in a Workflow. If the Workflow is a single-step workflow then the Workflow has just one Workflow Step entity. If the Workflow is a multi-step workflow then the Workflow has a Workflow Step entity for each step. All Workflow step specific attributes like parameters, and metadata about execution environment are captured by the Workflow Step entity.
      • MaterialEntity - A material entity is a physical entity that is spatially extended, exists as a whole at any point in time and has mass.
        • AnatomicalEntity - Biological entity that is either an individual member of a biological species or constitutes the structural organization of an individual member of a biological species.
        • Biospecimen - A Biospecimen is any natural material taken from a biological entity (usually a human) for testing, diagnostics, treatment, or research purposes. The Biospecimen is linked to the Individual from which the Biospecimen is derived.
        • CellLine - A cultured cell population that represents a genetically stable and homogenous population of cultured cells that shares a common propagation history.
        • Population - A population is a collection of individuals from the same taxonomic class living, counted or sampled at a particular site or in a particular area.
          • Cohort - A cohort is a collection of individuals that share a common characteristic/observation and have been grouped together for a research study/investigation.
          • Family - A domestic group, or a number of domestic groups linked through descent (demonstrated or stipulated) from a common ancestor, marriage, or adoption.
        • Sample - A sample is a limited quantity of something to be used for testing, analysis, inspection, investigation, demonstration, or trial use. A sample is prepared from a Biospecimen (isolate or tissue).
      • Person - A member of the species Homo sapiens.
        • Individual - An Individual is a Person who is participating in a Study.
          • Donor - A Donor is an Individual that participates in a research Study by donating a Biospecimen. The use of the Biospecimen is restricted to the consent provided by the Donor.
        • Member - Member of an Organization or a Committee.
        • User - A user in GHGA.
      • PlannedProcess - A process is an entity that exists in time by occurring or happening, has temporal parts and always involves and depends on some entity during the time it occurs.
        • AnalysisProcess - An analysis process is a process that describes how one or more Files, from a Study, are transformed to another set of Files via a Workflow. The analysis process also keeps track of the workflow metadata and the Agent that is running the Analysis.
        • DataTransformation - A data transformation technique used to analyze and interpret data to gain a better understanding of it.
          • Analysis - An Analysis is a data transformation that transforms input data to output data. The workflow used to achieve this transformation and the individual steps are also captured.
        • ExperimentProcess - An Experiment Process is a process that describes how a Sample is transformed to a File via an assay. The Experiment Process also keeps track of the Protocol used and the Agent that is running the experiment.
        • Investigation - Investigation is the process of carrying out a plan or procedure so as to discover fact or information about the object of study.
          • Experiment - An experiment is an investigation that consists of a coordinated set of actions and observations designed to generate data with the goal of verifying, falsifying, or establishing the validity of a hypothesis.
          • Study - Studies are experimental investigations of a particular phenomenon. It involves a detailed examination and analysis of a subject to learn more about the phenomenon being studied.
        • ResearchActivity - A planned process executed in the performance of scientific research wherein systematic investigations are performed to establish facts and reach new conclusions about phenomena in the world.
          • Project - Any specifically defined piece of work that is undertaken or attempted to meet a single requirement.
    • WorkflowParameter - A key/value pair that represents a parameter used in a Workflow Step.

    Mixins¶

    Slots¶

    • abstract - The abstract - a short blurb - associated with an entity.
    • accession - A unique identifier assigned to an entity for the sole purpose of referring to that entity in a global scope.
      • named thing➞accession
    • additional name - Additional name(s).
    • age - The age of a person.
      • individual➞age - Age of an Individual.
    • ancestry - Ancestral group to which a person belongs to.
      • individual➞ancestry - Ancestry of an Individual. Typically a value from Human Ancestry Ontology. For example, 'HANCESTRO:0005' signifies this Individual belongs to the 'European' ancestry.
    • category - The file category.
      • file➞category - The category for this file: Whole Genome Sequencing, Whole Exome Sequencing, etc.
    • checksum - The checksum for a file.
      • file➞checksum - The checksum for the file.
    • creation date - Timestamp (in ISO 8601 format) when the entity was created.
      • named thing➞creation date
    • description - Description of an entity.
      • data access policy➞description - A short description for the Data Access Policy.
      • dataset➞description
      • experiment➞description - A description for the Experiment.
      • project➞description - A detailed description about this Project.
      • protocol➞description - Detailed description of the Protocol.
      • sample➞description - Brief description of the Sample.
      • study➞description - A detailed description (abstract) that describes the goals of this Study.
    • email - Email of a person.
      • member➞email - The email of the Member.
    • ethnicity - Ethnicity of a person.
      • individual➞ethnicity - Ethnicity of an Individual.
    • family name - Last name.
    • file_index - The index file corresponding to the file.
      • file➞file_index - The index for this file. Commonly for BAM/VCF files.
    • format - The format of a file.
      • file➞format - The format of the file: BAM, SAM, CRAM, BAI, etc.
    • gender - The assemblage of properties that distinguish people on the basis of the societal role s expected for the two sexes. [NCI]
      • individual➞gender - Gender of an Individual.
    • geographical region - The geographical region linked to an entity.
      • individual➞geographical region - The geographical region where the Individual is located. Typically a value from Human Ancestry Ontology. For example, 'HANCESTRO:0047' signifies this Individual is from 'Western Europe' region.
    • given name - First name.
    • has agent
      • analysis process➞has agent - The Agent - a software, institution, or human - that is executing or responsible for executing the workflow.
      • experiment process➞has agent - The Agent - a software, institution, or human - that is executing or responsible for executing the Experiment Process.
    • has analysis - The analysis associated with an entity.
      • study➞has analysis - One or more Analysis entities associated with this Study.
    • has analysis process
      • analysis➞has analysis process - One or more Analysis Process entities associated with this Analysis.
    • has anatomical entity - Anatomical site associated with an entity.
      • biospecimen➞has anatomical entity - The Anatomical entity, that represents the site, from which the Biospecimen was retrieved. Typically, a concept from Uber-anatomy Ontology (UBERON). For example, 'UBERON:0008307' indicates that the Biospecimen was extracted from the 'Heart Endothelium' of an Individual.
    • has attribute - Key/value pairs corresponding to an entity.
      • named thing➞has attribute - Holds one or more Attribute entities that further characterizes this entity.
        • project➞has attribute - Custom key/value pairs that further characterizes this Project.
        • protocol➞has attribute - One or more attributes that further characterizes this Protocol.
        • study➞has attribute - Custom key/value pairs that further characterizes the Study.
    • has biospecimen
      • sample➞has biospecimen - The Biospecimen from which this Sample was prepared from.
    • has children - The children of an entity.
      • individual➞has children - One or more children for this Individual.
    • has data access committee - Data Access Committee associated with an entity.
      • data access policy➞has data access committee - The Data Access Committee linked to this policy.
    • has data access policy - Data Access Policy associated with an entity.
      • analysis dataset➞has data access policy - The Data Access Policy that applies to this Dataset.
      • experiment dataset➞has data access policy - The Data Access Policy that applies to this Dataset.
    • has data use condition - Data Use Condition entities that are associated with an entity.
      • data access policy➞has data use condition - Data Use Condition entities that are associated with the Data Access Policy.
    • has dataset
      • aggregate dataset➞has dataset - One or more Datasets that constitutes this Aggregate Dataset.
    • has disease - Disease concept that the entity is associated with.
      • biospecimen➞has disease - The Disease entity that is associated with this Biospecimen at the time of retrieval from the organism. Typically, a concept from Mondo Disease Ontology. For example, 'MONDO:0003742' indicates that the Individual - from which the Biospecimen was extracted from - suffers from 'Heart Fibrosarcoma'.
    • has experiment - The experiment associated with an entity.
      • analysis dataset➞has experiment - One or more Analysis entities that are referenced by this Dataset.
      • experiment dataset➞has experiment - One or more Experiment entities that are referenced by this Dataset.
      • study➞has experiment - One or more Experiment entities associated with this Study.
    • has experiment process
      • experiment➞has experiment process - One or more Experiment Processes entities associated with this Experiment.
    • has file - The file associated with an entity.
      • dataset➞has file - One or more File entities that collectively are part of this Dataset.
      • experiment➞has file - One or more Files entities that are generated as output of this Experiment.
    • has individual - The subject/individual associated with an entity.
      • biospecimen➞has individual - The Individual entity from which this Biospecimen was derived.
      • sample➞has individual - The Individual from which this Sample was derived from.
    • has input
      • analysis process➞has input - The input data File entities used in the Analysis Process.
      • analysis➞has input - The input data File entities used in the Analysis.
      • experiment process➞has input - The input to the Experiment Process. Usually a Sample entity.
    • has member - The members associated with a committee.
      • cohort➞has member - One or more Individuals that collectively define this Cohort.
      • data access committee➞has member - All the members that are part of this Data Access Committee.
      • family➞has member - One or more Individuals that collectively define this Family.
    • has output
      • analysis process➞has output - The output data File entities generated by the Analysis Process.
      • analysis➞has output - The output data File entities generated by this Analysis.
      • experiment process➞has output - The output of this Experiment Process. Usually a File entity.
    • has parameter
      • workflow step➞has parameter - One or more parameters that are associated with this Workflow Step.
    • has parent - The parent of an entity.
      • individual➞has parent - One or more parent for this Individual.
    • has phenotypic feature - Phenotypic feature concept that the entity is associated with.
      • biospecimen➞has phenotypic feature - The Phenotypic Feature entity that is associated with this Biospecimen at the time of retrieval from the organism. Typically, a concept from Human Phenotype Ontology. For example, 'HP:0100244' indicates that the Individual - from which the Biospecimen was extracted from - exhibits 'Fibrosarcoma' as one of its phenotype.
    • has protocol - The protocol associated with an entity.
      • experiment process➞has protocol - The Protocol entity used by this Experiment Process.
    • has publication - The Publication associated with an entity.
      • dataset➞has publication - One or more Publication entities associated with this Dataset.
      • project➞has publication - One or more Publication entities associated with this Project.
      • study➞has publication - One or more Publication entities associated with this Study.
    • has sample - The sample associated with an entity.
      • experiment➞has sample - The Sample entity associated with this Experiment.
    • has study - The study associated with an entity.
      • analysis dataset➞has study - One or more Study entities that are referenced by this Dataset.
      • analysis➞has study - The Study entity associated with this Analysis.
      • experiment dataset➞has study - One or more Study entities that are referenced by this Dataset.
      • experiment➞has study - The Study entity associated with this Experiment.
      • project➞has study - One or more Study entities associated with this Project.
    • has technology - The technology associated with an entity.
      • experiment➞has technology - The Technology entity associated with this Experiment.
    • has workflow
      • analysis➞has workflow - The Workflow entity associated with this Analysis.
    • has workflow step
      • analysis process➞has workflow step - Workflow Step entity that performs a set of operations on the input data Files to generate a set of output data Files.
    • id - An identifier that uniquely represents an entity.
      • named thing➞id - The internal unique identifier for an entity.
        • publication➞id - A PMID or DOI for the Publication.
    • key - The key of an attribute.
      • attribute➞key - The key for an attribute.
      • workflow parameter➞key - Key that represents the parameter name.
    • key type - The semantic type for the key of an attribute.
      • attribute➞key type - A semantic type that characterizes the attribute key. Usually this is a term from an ontology. For example, 'MAXO:0000616' indicates that the attribute is a measurement of oxygen saturation in the blood.
    • main contact - The person who is the main contact for a committee.
      • data access committee➞main contact - The main contact for the Data Access Committee.
    • modifier - Modifier for Data user permission.
      • data use condition➞modifier - Modifier for Data use permission. Preferable descendants of 'DUO:0000017 data use modifier'
    • name - The name for an entity.
      • data access committee➞name
      • experiment➞name - Name for the Experiment.
      • protocol➞name - Name of the Protocol.
      • sample➞name - Name of the Sample.
    • organization - The organization associated with a person.
      • member➞organization - The organization that the Member is part of.
    • parameters - Paramaters associated with a Workflow Step entity.
    • permission - Data use permission.
      • data use condition➞permission - Data use permission. Typically one or more terms from DUO. Preferably descendants of 'DUO:0000001 data use permission'.
    • policy text - The complete text for the Data Access Policy.
      • data access policy➞policy text - The terms of data use and policy verbiage should be captured here.
    • policy url - Alternative to pasting the Data Access Policy text.
      • data access policy➞policy url - URL for the policy, if available. This is useful if the terms of the policy is made available online at a resolvable URL.
    • proband
      • family➞proband - The Individual that is reported to have a disorder which results in the Family being brought into a Study.
    • role
      • user➞role - The role of the user
    • sex - The assemblage of physical properties or qualities by which male is distinguished from female.[NCI]
      • individual➞sex - Biological Sex of an Individual.
    • size - The size of a file in bytes.
      • file➞size - The file size in bytes.
    • telephone - The telephone number associated with a person.
      • member➞telephone - The telephone number of the Member.
    • title - The title that describes an entity.
      • dataset➞title
      • publication➞title - The title for the Publication.
      • study➞title - A descriptive title for the Study.
    • type - The type of an entity.
      • named thing➞type
        • dataset➞type
        • file➞type - The file type: Aligned Reads, Unaligned Reads, etc.
        • sample➞type - Type of sample: 'Case' or 'Control'.
      • study➞type - The type of Study. For example, "Cancer Genomics", "Epigenetics", "Exome Sequencing".
    • update date - Timestamp (in ISO 8601 format) when the entity was updated.
      • named thing➞update date
    • url - A URL to a resource.
      • protocol➞url - URL for the resource that describes this Protocol.
    • value - The value of an attribute
      • attribute➞value - The value for an attribute. Usually this is a numerical value (without the units).
      • workflow parameter➞value - Value corresponding to the the parameter key.
    • value type - The semantic type for the value of an attribute.
      • attribute➞value type - The value type that characterizes the attribute value. Usually this is a term from an ontology that describes how to interpret the value. For example, 'SIO:001413' indicates that the value is to be interpreted as a percentage.
    • vital status - Vital status of a person.
      • individual➞vital status - Vital status of an Individual.
    • xref - Alternate identifiers for an entity.
      • named thing➞xref - Holds one or more database cross references.
        • publication➞xref - One or more cross-references for this Publication.
        • sample➞xref - One or more cross-references for this Sample. For example, this Sample may have an EBI BioSamples accession or an EGA Sample accession.
    • year of birth - Year of Birth of a person.
      • individual➞year of birth - Year of Birth of an Individual.

    • case_control_enum - Enum to capture whether a Sample is to be considered as Case or Control.
