Ghga-Schema schema

The GHGA Core Schema is similar to the EGA Schema.

Classes

  • NamedThing - A databased entity, concept or class. This is a generic class that is the root of all the other classes.
    • DataAccessCommittee - A group of members that are delegated to grant access to one or more datasets after ensuring the minimum criteria for data sharing has been met, and request for data use does not raise ethical and/or legal concerns.
    • DataAccessPolicy - A data access policy specifies under which circumstances, legal or otherwise, a user can have access to one or more datasets belonging to one or more studies.
    • Dataset - A dataset is a collection of files that represents data generated from a particular sample.
    • Experiment - An experiment is an investigation that consists of a coordinated set of actions and observations designed to generate data with the goal of verifying, falsifying, or establishing the validity of a hypothesis.
    • Sample - A sample is a limited quantity of something to be used for testing, analysis, inspection, investigation, demonstration, or trial use. This could be either an individual, or sets of individuals from a population, or a portion of a substance like an isolate or tissue.
    • Study - Studies are experimental investigations of a particular phenomenon. It involves a detailed examination and analysis of a subject to learn more about the phenomenon being studied.

Mixins

Slots

  • abstract - The study abstract that describes the goals. Can also hold abstract from a publication related to the study.
    • study➞abstract
  • alias - Name which is matching the uploaded sample(s).
    • sample➞alias
  • biosample id - Specific sample ID from a submission to EBI BioSamples.
  • case or control - Is the sample case or control.
  • cell line - Which cell line was used.
  • custom tags - Key/value pairs corresponding to the study.
    • study➞custom tags
  • data access committees - One or more DACs associated with this Data Access Policy.
    • data access policy➞data access committees
  • data access policies - One or more DAPs associated with this dataset.
    • dataset➞data access policies
  • dataset description - A through description of this dataset.
    • dataset➞dataset description
  • dataset title - A descriptive title for this dataset.
    • dataset➞dataset title
  • dataset type - The type of dataset. For example, Whole Genome Sequencing, Whole Exome Sequencing, etc.
    • dataset➞dataset type
  • description - Description of experiment.
  • descriptive title - A descriptive title for the study.
    • study➞descriptive title
  • design name - The name of the experiment.
    • experiment➞design name
  • email - Email of the contact.
  • files - One or more files that are linked to an entity.
    • dataset➞files - One or more files that collectively make this dataset.
    • sample➞files - One or more files that are linked to this Sample.
  • gender - Gender of the subject.
    • sample➞gender
  • geographical region - Ethinicity.
  • id - An identifier that uniquely represents an entity.
  • instrument model - The instrument on which the data was generated.
    • experiment➞instrument model
  • library construction protocol - Description of how the library was constructed.
  • library layout - The layout of the library. Whether it is single-end, paired-end.
    • experiment➞library layout
  • library name - The library name.
  • library selection - Library selection process. For example, random, cDNA, etc.
    • experiment➞library selection
  • library source - The source of the library. For example, Metagenome, Transcriptome, etc.
    • experiment➞library source
  • library strategy - Library strategy. For example, ChIP-Seq, RNA-Seq, DNA-Seq, etc.
    • experiment➞library strategy
  • main contact - Is the contact the main contact for the DAC
    • data access committee➞main contact
  • name - The name of the point of contact for the DAC.
    • data access committee➞name
  • organism part - From where the sample was collected: Liver, Blood, Stool, etc.
  • organization - The organization of the point of contact for the DAC.
    • data access committee➞organization
  • phenotype or disease - Disease or phenotype that the subject is associated with.
    • sample➞phenotype or disease
  • policy - The complete text for the Data Access Policy.
    • data access policy➞policy
  • policy url - Alternative to pasting the Data Access Policy text.
  • pubmed id - The PubMed ID related to a study.
  • related study - The study to which the experiment belongs to.
    • experiment➞related study
  • short description - A short description for the Data Access Policy.
    • data access policy➞short description
  • short name - A short name for the study.
  • study type - The study type. For example, "Cancer Genomics", "Epigenetics", "Exome Sequencing".
    • study➞study type
  • subject id - The subject ID of the patient from experiment. Note that this ID should be non-identifiable.
    • sample➞subject id
  • telephone - The telephone number of the point of contact for the DAC.
    • data access committee➞telephone
  • title - Brief description of an entity.
    • data access committee➞title
    • sample➞title - Brief description of a sample.
